# Investigation: VitePress Configuration and Custom Theme

## Doc Reads

- No `llmdoc/` directory exists in this project.

## Code Sections

- `.vitepress/config.mjs` (`defineConfig`): Main VitePress site configuration including nav, sidebar, Vite plugins, proxy, and transformPageData hook.
- `.vitepress/theme/index.js`: Custom theme entry point; extends DefaultTheme, registers global components, applies custom CSS.
- `.vitepress/theme/Layout.vue`: Custom layout wrapping DefaultTheme.Layout with slot injections for diary features and visual effects.
- `.vitepress/theme/custom.css`: Comprehensive CSS overrides for brand colors, glassmorphism, hover animations, hero styling, sidebar collapse, and responsive layout.
- `utils/auto_sidebar.mjs` (`set_sidebar`): Auto-generates sidebar items by scanning filesystem directories recursively.
- `.vitepress/theme/components/DiaryList.vue`: Paginated article list fetching from `/diary-articles.json`.
- `.vitepress/theme/components/DiaryEditor.vue`: Markdown editor with live preview, image upload, and CRUD via `/api/articles`.
- `.vitepress/theme/components/DiaryActions.vue`: Edit/Delete buttons shown on diary article pages when authenticated.
- `.vitepress/theme/components/DiaryPageFooter.vue`: Back-to-list button and created/updated timestamps for diary articles.
- `.vitepress/theme/components/PasswordProtect.vue`: Client-side password gate for `/todo/` section (hardcoded password).
- `.vitepress/theme/components/MouseEffect.vue`: Custom duck-shaped cursor with sparkle/ripple particle effects.
- `.vitepress/theme/components/ScrollFx.vue`: Scroll progress bar and star particles spawned on scroll.
- `.vitepress/theme/components/BackToTop.vue`: Floating back-to-top button.
- `.vitepress/theme/components/ResizableSidebar.vue`: Drag handle to resize sidebar width (persisted to localStorage).
- `.vitepress/theme/components/SidebarToggle.vue`: Button to collapse/expand sidebar (persisted to localStorage).
- `.vitepress/theme/composables/diaryAuth.js` (`useDiaryAuth`): Session-based auth composable using `/api/login` endpoint.
- `.vitepress/theme/composables/copyCode.js` (`useCopyCodeFix`): Overrides VitePress copy-code behavior to strip shell prompts.
- `package.json`: Scripts (`docs:dev`, `docs:build`, `docs:preview`); deps: `vitepress ^1.6.3`, `gray-matter ^4.0.3`, `markdown-it ^14.1.1`.

## Report

**Conclusions:**

1. The site uses VitePress 1.6.x with a custom theme that extends the default theme rather than replacing it entirely.

2. `config.mjs` structure:
   - `title`: "gaga's blog"
   - `markdown.lineNumbers`: enabled
   - `vite.plugins`: custom `diaryListPlugin()` that generates `/public/diary-articles.json` at build time and watches diary files during dev
   - `vite.server.proxy`: proxies `/api` to `process.env.API_PROXY_TARGET || 'http://gaga0714.top'`
   - `transformPageData`: injects `diaryFileCreated` / `diaryFileUpdated` timestamps into diary and inter_code pages
   - `themeConfig.search`: local search with Chinese translations
   - `themeConfig.nav`: 6 top-level items (home, diary, interview topics dropdown, hand-written code, algorithm, todo)
   - `themeConfig.sidebar`: 12 sidebar groups auto-generated via `set_sidebar()`
   - `themeConfig.socialLinks`: GitHub link

3. Navigation architecture:
   - Top nav has a dropdown for "八股" (interview prep) with 9 sub-categories (HTML, CSS, JS, Vue3, React, Network, Engineering, TypeScript, Backend)
   - Sidebars are auto-generated by scanning the filesystem; `utils/auto_sidebar.mjs` reads directories recursively, filters by whitelist, and sorts diary/inter_code by file creation time (newest first)

4. Theme customization approach:
   - Single `custom.css` file overrides VitePress CSS custom properties and class selectors
   - Brand palette: warm amber/coral (`#c45a5a` primary, `#e8a87c` secondary, `#f0c5a0` accent)
   - Glassmorphism: `backdrop-filter: blur()` on navbar, sidebar, features, search modal
   - Background: radial gradient warm light spots; non-home pages add a fixed background image (`/bg-page.png`) with overlay
   - Hero: avatar float animation, tail-wag SVG pseudo-element, gradient text sweep on hover
   - Feature cards: top gradient bar, corner radial glow, lift on hover
   - Sidebar: collapsible via CSS class `html.sidebar-collapsed` toggled by SidebarToggle component
   - Content width: overridden to 860px (with aside) / 900px (without sidebar)
   - Respects `prefers-reduced-motion`

5. Custom theme entry (`index.js`):
   - Extends `DefaultTheme`
   - Replaces layout with custom `Layout.vue`
   - Registers 3 global components: `PasswordProtect`, `DiaryList`, `DiaryEditor` (used directly in markdown via component tags)
   - Calls `useCopyCodeFix()` in `enhanceApp`

6. Layout slots used:
   - `#doc-before`: DiaryPageFooter (back button + timestamps) and DiaryActions (edit/delete)
   - `#layout-bottom`: BackToTop, MouseEffect, ScrollFx, ResizableSidebar, SidebarToggle

7. Build/dev configuration:
   - Scripts: `docs:dev` (dev server), `docs:build` (production build), `docs:preview` (preview built site)
   - No custom Vite build options beyond the diary plugin and proxy
   - The diary plugin copies cover images from `assets/` to `public/covers/` during build

8. Auth model: Two separate auth mechanisms exist:
   - `/todo/` section: client-side hardcoded password in PasswordProtect.vue (session-only)
   - Diary CRUD: server-side auth via `/api/login` returning a token stored in sessionStorage

**Relations:**

- `config.mjs` imports `set_sidebar` from `utils/auto_sidebar.mjs` for sidebar generation
- `config.mjs` defines `diaryListPlugin()` which writes `public/diary-articles.json`; `DiaryList.vue` fetches this JSON at runtime
- `Layout.vue` imports and renders all 7 custom components in VitePress layout slots
- `index.js` imports `Layout.vue` and registers global components used in markdown files (`diary/index.md` uses `<DiaryList />`, `diary/edit.md` uses `<DiaryEditor />`)
- `DiaryEditor.vue`, `DiaryActions.vue`, and `DiaryList.vue` all consume `useDiaryAuth` composable
- `custom.css` styles reference component class names from both VitePress internals (`.VPHero`, `.VPFeature`, `.VPSidebar`, `.VPNavBar`) and custom components (`.back-to-top`, `.sidebar-resize-handle`)
- `SidebarToggle.vue` toggles `html.sidebar-collapsed` class; `custom.css` applies the transform/padding rules for that class

**Gaps:**

- No `vitepress` version lock file was inspected; actual resolved version unknown.
- The API backend (`gaga0714.top`) is external; its implementation is not in this repo.
- No test configuration or CI pipeline files were examined.
- The `todo/` section content structure was not explored beyond knowing it is password-protected.

**Result:**

The blog is a VitePress 1.6.x site with a warm-toned custom theme built on top of the default theme. Configuration lives in `.vitepress/config.mjs` with auto-generated sidebars, a custom Vite plugin for diary article metadata, and an API proxy for CRUD operations. The theme layer (`index.js` + `Layout.vue` + `custom.css`) adds glassmorphism styling, interactive hover animations, a custom duck cursor, scroll particle effects, resizable/collapsible sidebar, and a full diary CMS (list, editor, auth). All visual customizations are done through CSS variable overrides and scoped/global styles rather than forking VitePress internals.
