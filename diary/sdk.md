# SDK
SDK 是 Software Development Kit 的缩写，中文常叫 软件开发工具包。它的核心作用是让你更快、更规范地基于某个平台或某个服务去开发应用，而不是从零开始对接所有细节。

## SDK 通常包含什么
1. API 接口与调用方式
你在代码里真正会调用的函数、类、方法，比如创建对象、发请求、读写数据等。

2. 依赖库与运行组件
比如动态链接库、Jar 包、NuGet 包、Python 包等，让功能在本地能跑起来。

3. 文档与示例代码
说明怎么初始化、鉴权、传参、处理返回值，以及常见场景的 Demo。

4. 工具与调试能力
可能包含命令行工具、日志开关、错误码表、Mock 工具、测试用例等。

## 为什么需要 SDK
1. 封装复杂性
把协议细节、签名鉴权、重试、序列化等通用逻辑封起来，你只关心业务调用。

2. 提高一致性与稳定性
统一错误处理、超时策略、版本兼容，减少你自己写一套导致的不一致。

3. 提升开发效率
少写很多胶水代码，照着示例就能接入。

## 和 API 的区别
API 是能力本身，通常是服务端提供的接口定义。
SDK 是把这些 API 用某种语言封装成你更容易用的代码工具包，并且往往附带鉴权、签名、重试、分页、类型定义等工程化能力。

举个贴近你前面 IFC 的类比
如果你要生成 IFC
Schema 规范相当于规则
SDK 类似 IfcOpenShell 这种库，帮你用代码去创建 IfcProject、IfcWall、IfcRelContainedInSpatialStructure 这些对象，并负责写成合法的 .ifc 文件格式。

# Schema是什么
IFC Schema 是 buildingSMART 定义的实体模型规范
比如 IfcWall IfcProject IfcRelAggregates 这些实体有哪些属性、属性的类型、以及它们之间允许的关系
你用 IfcOpenShell 生成 IFC 文件时，创建的每个实体都必须符合 IFC Schema，否则文件可能不合法或别的软件读不出来

Schema 是数据的规范说明书
数据是按说明书写出来的实例
