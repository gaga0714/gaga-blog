# Startup Reading Order

Read these documents in order when starting work on gaga-blog:

1. `llmdoc/must/project-basics.md` - Identity, tech stack, content organization, key commands
2. `llmdoc/overview/project-overview.md` - Project boundaries and major areas
3. `llmdoc/architecture/vitepress-blog.md` - VitePress config, theme system, content discovery
4. `llmdoc/architecture/backend-cms.md` - Express API, authentication, git integration

## When to Read More

- **Deployment issues**: `llmdoc/reference/deployment.md`
- **Content gaps or doc structure questions**: `llmdoc/memory/doc-gaps.md`

## Escalation

If the startup docs don't answer your question:
1. Check if a guide exists in `llmdoc/guides/`
2. Check reference docs in `llmdoc/reference/`
3. Search the codebase directly
4. Propose a new doc in `llmdoc/memory/doc-gaps.md`
