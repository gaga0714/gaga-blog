# 随记功能 - 修改计划

> 在导航栏"鸭棚子"后面加一个"胡言乱语(新版,即随记)",可在线发布、编辑、删除文章,带分页列表展示;迁移现有 30+ 篇 diary 文章。

## 一、最终确认的产品规格

- 导航顺序:`鸭棚子` → **`胡言乱语`(新)** → `八股` → `手撕` → `算法` → `人生是旷野`
- `/diary/` 路径保留,但内容从"散落 md 列表 + 自动侧栏"变成"卡片式分页列表 + 客户端分页"
- 每页 10 条,按 `updated` 倒序;每条:首图缩略 + 标题 + 时间 + 摘要
- 点卡片进文章详情页(普通 VitePress 渲染),无显式返回按钮(用浏览器后退)
- 登录态(密码 → token 存 sessionStorage)下可见"发布新随记";文章详情页可见"编辑 / 删除"
- 双栏 markdown 编辑器,实时预览,支持图片上传
- 后端 Node + Express,接收发布请求 → 写 md + 推图片 → `git push` 到 main → CI 重建部署(30–60 秒生效)
- 现有 30+ 篇 diary 文章一次性迁移:加 frontmatter(title / created / updated),URL 不变

---

## 二、技术决策一览

| 项 | 选择 |
| --- | --- |
| 在线加文章 | 后端 API 写入仓库,CI 重新部署 |
| 后端语言 | Node + Express |
| 后端部署 | 跑在现有那台服务器(走 git push) |
| 发布权限 | 密码(只你一个人) |
| 凭证存储 | sessionStorage(关浏览器失效) |
| 编辑器 | 双栏实时预览 |
| 图片 | 上传到仓库 `public/diary-images/` |
| 标题/元数据 | 独立标题输入框 + frontmatter 存时间 |
| 编辑/删除 | 都支持 |
| 分页 | 客户端分页,URL 保持 `/diary/` |
| 返回按钮 | 不加,用浏览器后退 |
| 列表展示 | 标题 + 时间 + 摘要 + 首图 |
| 文件名 | 时间戳 + 随机后缀,如 `2026-05-22-a3f9.md` |
| 迁移策略 | 一步到位(迁移 + 改导航) |

## 三、修改步骤

### 阶段 0:服务器准备(手动操作,我给清单)

需要在服务器上完成,会写在 `server/README.md`:

1. 选个目录(比如 `/srv/blog-repo`),`git clone` 仓库
2. 服务器生成 SSH key,公钥配到 GitHub Deploy keys(勾上 write),让服务器能 `git push`
3. 装 Node 18+、`pm2`
4. 仓库根目录 `server/` 下 `npm i`,`pm2 start ecosystem.config.cjs`
5. 编辑 nginx 配置:`location /api/ { proxy_pass http://127.0.0.1:3001; }` + reload
6. 环境变量(`server/.env`):
   - `BLOG_PASSWORD`(发布密码)
   - `JWT_SECRET`
   - `REPO_PATH=/srv/blog-repo`
   - `PORT=3001`
   - `GIT_USER_NAME`、`GIT_USER_EMAIL`

### 阶段 1:后端服务(新建 `server/` 目录)

新文件:

- `server/index.mjs`:Express 主入口
  - `POST /api/login`:校验密码 → 返回 JWT
  - `POST /api/articles`:验 JWT → body `{title, content}` → 生成 `<YYYY-MM-DD>-<4位hex>.md` → 写到 `<REPO_PATH>/diary/`,带 frontmatter `title` / `created` / `updated` → `git add + commit + push`
  - `PUT /api/articles/:slug`:验 JWT → 读旧 md → 替换正文 → 更新 `updated` → push
  - `DELETE /api/articles/:slug`:验 JWT → `git rm` → push
  - `POST /api/upload`:验 JWT → multer 收图片 → 存到 `<REPO_PATH>/public/diary-images/<YYYY>/<uuid>.<ext>` → `git add`(暂不 push,随下一次文章 push 一起) → 返回 `/diary-images/<YYYY>/<uuid>.<ext>`
- `server/git.mjs`:封装 `simple-git` 调用
- `server/auth.mjs`:JWT 签发 + 校验中间件
- `server/.env.example`、`server/package.json`、`server/ecosystem.config.cjs`、`server/README.md`(部署指南)

### 阶段 2:迁移现有 diary 文章(一次性脚本)

新文件 `scripts/migrate-diary.mjs`:

- 遍历 `diary/*.md`,跳过 `index.md`
- 已有 frontmatter 的跳过
- 每个文件:
  - `title`:取第一行 `# xxx`,没有就用文件名(去 `.md`)
  - `created`:`git log --diff-filter=A --format=%cI -- <file> | tail -1`
  - `updated`:`git log -1 --format=%cI -- <file>`
- 写回原文件(frontmatter 加在最顶部)
- 本地跑一次 → commit + push,完成迁移

### 阶段 3:构建时生成文章 JSON

修改 `.vitepress/config.mjs`,加 vite plugin(或 `buildStart` 钩子):

- 启动时扫描 `diary/*.md`(跳过 `index.md` 和 `edit.md`)
- 每篇解析:`title`、`updated`、摘要(去掉 frontmatter 和图片后的前 ~100 字)、首图(第一个 `![](...)` 的 src)、`slug`(文件名去扩展名)
- 按 `updated` 倒序输出到 `public/diary-articles.json`
- dev 模式下监听 `diary/*.md` 改动重新生成

### 阶段 4:列表页(重写 `diary/index.md`)

`diary/index.md` 改成 VitePress 自定义页面:

```md
---
layout: page
title: 胡言乱语
---

<DiaryList />
```

新建 `.vitepress/theme/components/DiaryList.vue`:

- `onMounted` fetch `/diary-articles.json`
- 状态:`page`(从 URL `?page=N` 同步),每页 10 条
- 卡片样式:左 120×120 首图(无图就用占位渐变),右标题 + 时间(`xx 天前` + `2026-05-22 14:30` hover)+ 摘要
- 顶部右侧:"发布新随记"按钮(仅登录态)→ 跳 `/diary/edit`
- 卡片右下角:"编辑 / 删除"小图标(仅登录态)
- 底部分页器:`< 1 2 3 ... N >`

`.vitepress/theme/index.js` 注册 `DiaryList` 全局组件。

### 阶段 5:发布/编辑器页面

新建 `diary/edit.md`:

```md
---
layout: page
title: 写随记
---

<DiaryEditor />
```

新建 `.vitepress/theme/components/DiaryEditor.vue`:

- 未登录:显示密码输入 → POST `/api/login`,拿到 token 存 sessionStorage
- 已登录:
  - 顶部"标题"输入框
  - 双栏:左 textarea(markdown),右实时预览(markdown-it 渲染)
  - 工具栏:"插入图片"(选文件 → POST `/api/upload` → 在光标处插入 `![](url)`)
  - URL `?slug=xxx` 时进入编辑模式:加载现有文章(fetch md 原文,去掉 frontmatter 灌进 textarea)
  - 底部:"发布"(POST)/"保存"(PUT) + 取消
  - 编辑模式下额外有"删除"按钮(二次确认 → DELETE)

### 阶段 6:文章详情页(轻改)

文章详情页本来就是 VitePress 自动渲染 md。需要做的:

- `transformPageData` 里给 `diary/*` 注入 `updated` 时间(读 frontmatter),页面顶部展示
- 新建 `.vitepress/theme/components/DiaryActions.vue`:登录态时显示"编辑 / 删除"按钮
- 通过 `doc-before` slot(沿用现有 `DiaryPageFooter` 的注入方式)在文章页插入。检测路径在 `/diary/` 下且不是 `index` 和 `edit` 才显示

### 阶段 7:导航和侧边栏

`.vitepress/config.mjs`:

- nav 数组:`{ text: '胡言乱语', link: '/diary/' }` 从尾部移到第二位(鸭棚子 之后,八股 之前)
- sidebar 配置删掉 `/diary/` 那一项
- `utils/auto_sidebar.mjs` 不动

### 阶段 8:本地验证

跑 `npm run docs:dev`,确认:

- 顶部导航顺序正确
- `/diary/` 列表展示、分页、首图、摘要都正常
- 旧 diary 文章点进去能渲染、有更新时间
- `/diary/edit` 编辑器交互正常(后端没起的情况下登录会失败,符合预期)

后端起来后,你自己上服务器再验一遍发布/编辑/删除/上传。

## 五、工作量预估和风险

- 后端 + 前端代码:大约 1500–2000 行
- 手动部署:大约 30–60 分钟
- **最大风险点**:服务器 git push 的 SSH key 配置,配不对后端会报错(后端会加详细错误日志)
- **中等风险点**:图片上传到仓库会让仓库慢慢变大,长期可能要考虑迁到对象存储
