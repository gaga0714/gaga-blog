# 握手挥手

客户端fin_wait1 服务器close_wait 客户端fin_wait2 服务器last_ack 客户端time_wait 服务器close客户端close
在 TCP 的四次挥手里，客户端在发出最后一个 ACK 后进入 TIME_WAIT，并等待 2×MSL。这段时间内：
如果服务器没收到这个 ACK，它会重发 FIN。客户端仍在 TIME_WAIT，一旦又收到 FIN，就再回一个 ACK。
如果在 2×MSL 内没有再收到 FIN，客户端就推断服务器已经收到了先前的 ACK，并安全地把连接彻底关掉。

# websocket
没有ws之前是轮询：不停的向服务器发送http请求问有没有数据，浪费大量带宽和cpu资源
全双工，二进制帧，ws明文80，wss密文443
请求：
```shell
GET /chat HTTP/1.1
Host: server.example.com
Connection: Upgrade
Upgrade: websocket
Origin: http://example.com
Sec-WebSocket-Key: dsdnfisughnerfjferfe==
Sec-WebSocket-Protocol: chat,superchat
Sec-WebSocket-Version: 13
```

响应：
```shell
HTTP/1.1 101 Switching Protocols
Connection: Upgrade
Upgrade: websocket
Sec-WebSocket-Accept: sdfgngjrenfernfje=
Sec-Websocket-Protocol: chat
```

应用场景：弹幕，媒体聊天，协同编辑，基于位置的应用，体育实况更新，股票基金报价实时更新
