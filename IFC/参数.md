placement
三元组 (x, y, z),快速把构件放到某个点上，但不考虑旋转。
在代码里会被转换成 IfcLocalPlacement + IfcAxis2Placement3D，其中只设置原点，不显式设置朝向。
matrix
4x4 的齐次变换矩阵，包含平移和旋转等变换。
在代码里通过 edit_object_placement 写入产品的放置。

predefined_type
IFC 里每类构件通常有一个枚举 PredefinedType，用来细分子类型。

space
一个 ifcopenshell.entity_instance，通常是 IfcSpace。
表示把这个构件归属到某个房间或空间单元下。
做空间级别的查询、统计或逻辑归属。
一般对实体构件有效，对 Opening 通常不建议挂。

storey
一个 ifcopenshell.entity_instance，通常是 IfcBuildingStorey。
表示把这个构件归属到某个楼层下。
绝大多数实体构件都应至少归属到某个 storey，方便模型结构化与浏览。

material
一个 ifcopenshell.entity_instance，可以是IfcMaterial,IfcMaterialLayerSet 或 IfcMaterialLayerSetUsage,IfcMaterialConstituentSet 等
表达构件材质信息，影响算量、分析、渲染、规则校验等。
门窗通常有材料，墙板柱梁也常有材料。Opening 一般不赋材质。

representation
一个 ifcopenshell.entity_instance，通常是 IfcProductDefinitionShape，内部包含一个或多个 IfcShapeRepresentation。
它承载几何形体信息，比如拉伸体、BRep、网格等。
让构件“有形状可显示”。没有 representation 的产品通常只有语义壳子，在软件里可能不可见或仅显示为点。

psets
一个字典 dict，结构通常是{ pset_name: { property_name: value, ... }, ... }
挂业务属性或扩展属性集。比如尺寸、编码、业务字段、算法标注字段、筛选字段等。

relate_element
一个 ifcopenshell.entity_instance，通常是被开洞的承载构件，例如 IfcWall IfcSlab IfcBeam 等。
用于在 create_opening 里调用 add_opening_to_element。
IFC 语义:建立 void 关系，通常对应 IfcRelVoidsElement：承载构件被 Opening 挖空。

filling_element
一个 ifcopenshell.entity_instance，通常是 IfcDoor 或 IfcWindow。
用于在 create_opening 里调用 add_filling_to_opening。
IFC 语义:建立填充关系，通常对应 IfcRelFillsElement：门窗填充该 Opening。

opening
一个 ifcopenshell.entity_instance，通常就是 IfcOpeningElement。
它出现在 create_door 或 create_window 的参数里时，含义是把门或窗作为 filling 元素关联到这个 opening 上，等价于在门窗函数内部调用 add_filling_to_opening。



placement 或 matrix 负责放置
predefined_type 负责语义细分
space storey 负责空间结构归属
material 负责材质
representation 负责几何
psets 负责属性扩展
relate_element 负责 Opening 挖到谁身上
filling_element 或 opening 负责门窗填到哪个洞口里

（1）项目与样式初始化 API
1 create_empty_project name
作用 创建一个空 IFC 项目文件，包含 IfcProject 单位设置 几何上下文 Model 与 Body，并按 DEFAULT_COLORS 预置材质与颜色样式
输入 name 项目名称
输出 ifcopenshell.file

2 create_single_building_project name
作用 在空项目基础上创建 Site 与 Building 并挂接到 Project
输入 name 项目名称
输出 ifcopenshell.file

3 create_single_storey_building_project name
作用 在单建筑项目基础上创建一个默认楼层 Storey
输入 name 项目名称
输出 ifcopenshell.file

4 add_element_color model name rgb transparency body_context
作用 创建 IfcMaterial 与可视化样式 IfcSurfaceStyleShading，并绑定到 Body 上下文，用于后续构件按类别着色
输入
model IFC 文件
name 类别名，例如 WALL PILLAR
rgb 颜色字典
transparency 透明度
body_context 可选，默认从模型获取 Body
输出 IfcMaterial 或 None

空间层级 API

作用 创建 IfcSite 并通过聚合关系挂到 IfcProject

placement 简单平移

storey 可选，默认取模型中的 IfcBuildingStorey

构件生成 API
1 create_covering model name placement matrix predefined_type space storey material representation psets
作用 创建 IfcCovering 装饰面，支持放置，挂接到 space 或 storey，绑定材质 几何 属性集

2 create_opening model name predefined_type placement matrix representation psets relate_element filling_element
作用 创建 IfcOpeningElement 洞口，支持放置 几何 属性集，并可选自动建立

关系构建 API
1 add_opening_to_element model opening element
作用 创建 IfcRelVoidsElement，表示 opening 挖在 element 上，通常 element 是墙
特点 内部会处理重复关系与清理旧关系，并校正 opening 的 placement
输出 无显式返回

2 add_filling_to_opening model opening element
作用 创建 IfcRelFillsElement，表示 element 填充到 opening，通常 element 是门或窗
特点 内部会处理重复关系与清理旧关系
输出 无显式返回

输出 IfcPropertySet 实例

2 edit_pset model pset properties
作用 编辑已有属性集写入属性
输出 pset

3 remove_pset model product pset
